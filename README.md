# elevator
电梯群控调度算法设计与实现
### 1. 初始设置
- 楼层高度（1-25）、电梯数量（1-8）、模拟时间、电梯限额配置

### 2. 调度模式设置
- 支持 Up-peak、Down-peak、Random 三种模式
- 四项指标权值固定，不可调整

### 3. 总控制台
- 生成随机数据 / 导入自定义数据
- 开始 / 暂停 / 退出控制

### 4. 仿真结果可视化
- 电梯位置实时动态演示
- 运行方向、载客状态显示

### 5. 输出仿真结果
- 平均候梯时间、乘梯时间统计
- 启停次数、空转时间、拥挤度分析

---

**核心算法**：蚁群算法（ACO）  
**可视化框架**：MFC

## 算法实现

### 1. 综合评价指标体系
- **候梯时间**：乘客发出请求到进入电梯的时间
- **乘梯时间**：进入电梯到到达目的楼层的时间
- **能耗**：电梯启停次数与运行距离的加权计算
- **拥挤度**：轿厢内人数占限额比例的实时统计

### 2. 蚁群算法核心
- 信息素矩阵初始化
- 状态转移概率计算
- 信息素动态更新（全局最优 + 局部更新）
- 收敛判断与终止条件

### 3. 多场景性能分析
- 电梯数量对比：1-8 部
- 楼层高度对比：1-25 层
- 电梯限载：n人

### 4. 动态仿真引擎
- 离散事件时间推进
- 乘客到达事件处理
- 电梯到达与上下客逻辑
- 实时状态同步

### 5. 数据读取
- 从文件导入乘客请求数据
- 支持 Up-peak / Down-peak / Random 三种调度模式
- 支持随机数据生成

### 6. 结果输出模块
- 平均候梯时间、乘梯时间计算
- 电梯启停次数统计
- 空转时间记录
- 拥挤度分布分析
